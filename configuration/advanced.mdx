## Runtime Fallback

Auto-switches to backup models when API errors occur.

### Basic Configuration

**Enable with defaults:**
```jsonc
{
  "runtime_fallback": true
}
```

**Disable:**
```jsonc
{
  "runtime_fallback": false
}
```

### Advanced Configuration

Full control over fallback behavior:

```jsonc
{
  "runtime_fallback": {
    "enabled": true,
    "retry_on_errors": [400, 429, 503, 529],
    "max_fallback_attempts": 3,
    "cooldown_seconds": 60,
    "timeout_seconds": 30,
    "notify_on_fallback": true
  }
}
```

### Configuration Options

<ParamField path="runtime_fallback.enabled" type="boolean" default={false}>
  Enable runtime fallback system.
</ParamField>

<ParamField path="runtime_fallback.retry_on_errors" type="number[]" default="[400, 429, 503, 529]">
  HTTP status codes that trigger fallback. Also handles classified provider key errors.
  
  **Common codes:**
  - `400` — Bad request / invalid key
  - `429` — Rate limit exceeded
  - `503` — Service unavailable
  - `529` — Overloaded
</ParamField>

<ParamField path="runtime_fallback.max_fallback_attempts" type="number" default={3}>
  Maximum fallback attempts per session (1–20).
  
  ```jsonc
  "max_fallback_attempts": 5
  ```
</ParamField>

<ParamField path="runtime_fallback.cooldown_seconds" type="number" default={60}>
  Seconds before retrying a failed model.
  
  ```jsonc
  "cooldown_seconds": 120  // Wait 2 minutes
  ```
</ParamField>

<ParamField path="runtime_fallback.timeout_seconds" type="number" default={30}>
  Seconds before forcing next fallback when provider hangs.
  
  **Set to `0` to disable timeout-based escalation:**
  ```jsonc
  "timeout_seconds": 0  // Only error-based fallback
  ```
</ParamField>

<ParamField path="runtime_fallback.notify_on_fallback" type="boolean" default={true}>
  Show toast notification on model switch.
  
  ```jsonc
  "notify_on_fallback": false  // Silent fallback
  ```
</ParamField>

### Per-Agent Fallback Models

Define fallback models for specific agents:

```jsonc
{
  "runtime_fallback": true,
  "agents": {
    "sisyphus": {
      "model": "anthropic/claude-opus-4-6",
      "fallback_models": ["openai/gpt-5.2", "google/gemini-3-pro"]
    },
    "hephaestus": {
      "model": "openai/gpt-5.3-codex",
      "fallback_models": "github-copilot/gpt-5.2"  // Single fallback
    }
  }
}
```

### Per-Category Fallback Models

Define fallback models for categories:

```jsonc
{
  "runtime_fallback": true,
  "categories": {
    "ultrabrain": {
      "model": "openai/gpt-5.3-codex",
      "fallback_models": ["anthropic/claude-opus-4-6", "google/gemini-3-pro"]
    },
    "quick": {
      "model": "anthropic/claude-haiku-4-5",
      "fallback_models": ["opencode/gpt-5-nano"]
    }
  }
}
```

### Complete Example

```jsonc
{
  "runtime_fallback": {
    "enabled": true,
    "retry_on_errors": [400, 429, 503, 529],
    "max_fallback_attempts": 5,
    "cooldown_seconds": 90,
    "timeout_seconds": 45,
    "notify_on_fallback": true
  },
  "agents": {
    "sisyphus": {
      "model": "anthropic/claude-opus-4-6",
      "fallback_models": [
        "kimi-for-coding/k2p5",
        "openai/gpt-5.2",
        "google/gemini-3-pro"
      ]
    }
  }
}
```

## Hashline Edit

Replaces the built-in `Edit` tool with a **hash-anchored version** using `LINE#ID` references to prevent stale-line edits.

<ParamField path="hashline_edit" type="boolean" default={true}>
  Enable hashline edit tool.
  
  **Enabled by default:**
  ```jsonc
  {
    // hashline_edit is enabled by default
  }
  ```
  
  **Disable:**
  ```jsonc
  {
    "hashline_edit": false
  }
  ```
</ParamField>

### How It Works

1. **Read tool** annotates output with hash anchors:
   ```
   1#a3f2  import { z } from "zod"
   2#9b1e  import { config } from "./config"
   3#7c4d  
   4#2e8f  export function load() {
   ```

2. **Edit tool** uses anchors instead of line numbers:
   ```
   oldString: "2#9b1e  import { config } from \"./config\""
   newString: "import { loadConfig } from \"./config\""
   ```

3. **Protection:** Edit fails if anchor doesn't match (file changed)

### Companion Hooks

When enabled, two hooks activate:

- `hashline-read-enhancer` — Annotates Read output
- `hashline-edit-diff-enhancer` — Shows diffs

**Disable individual hooks:**
```jsonc
{
  "hashline_edit": true,
  "disabled_hooks": [
    "hashline-read-enhancer"  // Keep edit, remove read annotations
  ]
}
```

## Background Tasks

Control parallel agent execution and concurrency limits.

```jsonc
{
  "background_task": {
    "defaultConcurrency": 5,
    "staleTimeoutMs": 180000,
    "providerConcurrency": {
      "anthropic": 3,
      "openai": 5,
      "google": 10,
      "opencode": 20,
      "zai-coding-plan": 10
    },
    "modelConcurrency": {
      "anthropic/claude-opus-4-6": 2,
      "opencode/gpt-5-nano": 50
    }
  }
}
```

### Configuration Options

<ParamField path="background_task.defaultConcurrency" type="number">
  Maximum concurrent tasks across all providers.
  
  ```jsonc
  "defaultConcurrency": 10
  ```
</ParamField>

<ParamField path="background_task.staleTimeoutMs" type="number" default={180000}>
  Interrupt tasks with no activity (minimum: 60000ms = 1 minute).
  
  ```jsonc
  "staleTimeoutMs": 300000  // 5 minutes
  ```
</ParamField>

<ParamField path="background_task.providerConcurrency" type="object">
  Per-provider concurrency limits. Key = provider name.
  
  ```jsonc
  "providerConcurrency": {
    "anthropic": 3,
    "openai": 5,
    "google": 10
  }
  ```
</ParamField>

<ParamField path="background_task.modelConcurrency" type="object">
  Per-model concurrency limits. Key = `provider/model`. Overrides provider limits.
  
  ```jsonc
  "modelConcurrency": {
    "anthropic/claude-opus-4-6": 2,
    "anthropic/claude-haiku-4-5": 10
  }
  ```
</ParamField>

**Priority:** `modelConcurrency` > `providerConcurrency` > `defaultConcurrency`

## Experimental Features

Cutting-edge features that may change or be removed.

```jsonc
{
  "experimental": {
    "aggressive_truncation": false,
    "truncate_all_tool_outputs": false,
    "auto_resume": false,
    "disable_omo_env": false,
    "task_system": false,
    "hashline_edit": false,
    "model_fallback_title": false,
    "plugin_load_timeout_ms": 10000,
    "safe_hook_creation": true,
    "dynamic_context_pruning": {
      "enabled": false,
      "notification": "detailed",
      "turn_protection": { "enabled": true, "turns": 3 },
      "protected_tools": ["task", "todowrite", "todoread"],
      "strategies": {
        "deduplication": { "enabled": true },
        "supersede_writes": { "enabled": true, "aggressive": false },
        "purge_errors": { "enabled": true, "turns": 5 }
      }
    }
  }
}
```

### Feature Flags

<ParamField path="experimental.aggressive_truncation" type="boolean" default={false}>
  Aggressively truncate when token limit exceeded.
</ParamField>

<ParamField path="experimental.truncate_all_tool_outputs" type="boolean" default={false}>
  Truncate all tool outputs, not just whitelisted tools.
  
  <Note>
    Tool output truncator is enabled by default. Disable via `disabled_hooks`.
  </Note>
</ParamField>

<ParamField path="experimental.auto_resume" type="boolean" default={false}>
  Auto-resume after thinking block recovery.
</ParamField>

<ParamField path="experimental.disable_omo_env" type="boolean" default={false}>
  Disable auto-injected `<omo-env>` block (date/time/locale). Improves cache hit rate.
</ParamField>

<ParamField path="experimental.task_system" type="boolean" default={false}>
  Enable Sisyphus task system for cross-session tracking.
</ParamField>

<ParamField path="experimental.model_fallback_title" type="boolean" default={false}>
  Append fallback model info to session title when runtime fallback occurs.
</ParamField>

<ParamField path="experimental.plugin_load_timeout_ms" type="number" default={10000}>
  Timeout in ms for `loadAllPluginComponents` during config handler init (minimum: 1000).
  
  ```jsonc
  "plugin_load_timeout_ms": 15000  // 15 seconds
  ```
</ParamField>

<ParamField path="experimental.safe_hook_creation" type="boolean" default={true}>
  Wrap hook creation in try/catch to prevent plugin crashes.
  
  **Disable for debugging:**
  ```jsonc
  "safe_hook_creation": false
  ```
</ParamField>

### Dynamic Context Pruning

Auto-prune old tool outputs to manage context window.

<ParamField path="experimental.dynamic_context_pruning.enabled" type="boolean" default={false}>
  Enable dynamic context pruning.
</ParamField>

<ParamField path="experimental.dynamic_context_pruning.notification" type="enum" default="detailed">
  Pruning notifications: `off`, `minimal`, `detailed`
</ParamField>

<ParamField path="experimental.dynamic_context_pruning.turn_protection" type="object">
  Protect recent turns from pruning.
  
  ```jsonc
  "turn_protection": {
    "enabled": true,
    "turns": 3  // Protect last 3 turns (1-10)
  }
  ```
</ParamField>

<ParamField path="experimental.dynamic_context_pruning.protected_tools" type="string[]">
  Tools whose outputs are never pruned.
  
  ```jsonc
  "protected_tools": [
    "task",
    "todowrite",
    "todoread",
    "lsp_rename",
    "session_read",
    "session_write",
    "session_search"
  ]
  ```
</ParamField>

<ParamField path="experimental.dynamic_context_pruning.strategies" type="object">
  Pruning strategies.
  
  **Deduplication:**
  ```jsonc
  "deduplication": { "enabled": true }
  ```
  
  **Supersede writes:**
  ```jsonc
  "supersede_writes": {
    "enabled": true,
    "aggressive": false  // Prune ANY write if subsequent read exists
  }
  ```
  
  **Purge errors:**
  ```jsonc
  "purge_errors": {
    "enabled": true,
    "turns": 5  // Prune errored inputs after 5 turns
  }
  ```
</ParamField>

## Sisyphus Task System

Enable cross-session task tracking:

```jsonc
{
  "sisyphus": {
    "tasks": {
      "enabled": false,
      "storage_path": ".sisyphus/tasks",
      "claude_code_compat": false
    }
  }
}
```

<ParamField path="sisyphus.tasks.enabled" type="boolean" default={false}>
  Enable Sisyphus Tasks system.
</ParamField>

<ParamField path="sisyphus.tasks.storage_path" type="string" default=".sisyphus/tasks">
  Storage path relative to project root.
</ParamField>

<ParamField path="sisyphus.tasks.claude_code_compat" type="boolean" default={false}>
  Enable Claude Code path compatibility mode.
</ParamField>

## Notification Settings

Force-enable session notifications:

```jsonc
{
  "notification": {
    "force_enable": true
  }
}
```

<ParamField path="notification.force_enable" type="boolean" default={false}>
  Force session-notification even if external notification plugins are detected.
</ParamField>

## Tmux Integration

Run background subagents in separate tmux panes.

<Warning>
  Requires running inside tmux with `opencode --port <port>`.
</Warning>

```jsonc
{
  "tmux": {
    "enabled": true,
    "layout": "main-vertical",
    "main_pane_size": 60,
    "main_pane_min_width": 120,
    "agent_pane_min_width": 40
  }
}
```

<ParamField path="tmux.enabled" type="boolean" default={false}>
  Enable tmux pane spawning.
</ParamField>

<ParamField path="tmux.layout" type="enum" default="main-vertical">
  Tmux layout: `main-vertical`, `main-horizontal`, `tiled`, `even-horizontal`, `even-vertical`
</ParamField>

<ParamField path="tmux.main_pane_size" type="number" default={60}>
  Main pane percentage (20–80).
</ParamField>

<ParamField path="tmux.main_pane_min_width" type="number" default={120}>
  Minimum main pane columns.
</ParamField>

<ParamField path="tmux.agent_pane_min_width" type="number" default={40}>
  Minimum agent pane columns.
</ParamField>
