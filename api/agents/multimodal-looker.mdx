## Overview

Multimodal Looker is a specialized agent for analyzing media files that cannot be read as plain text. It interprets PDFs, images, diagrams, and other visual content to extract specific information or provide summaries.

**Identity**: Media interpretation specialist that saves context tokens by analyzing files and returning only requested information.

<ParamField path="model" type="string" default="kimi-k2.5-free">
  Multimodal-capable model optimized for vision and document analysis
</ParamField>

<ParamField path="mode" type="string" default="subagent">
  Invoked by other agents when media analysis is needed
</ParamField>

<ParamField path="temperature" type="number" default="0.1">
  Very low temperature for consistent, accurate extraction
</ParamField>

## When to Use Multimodal Looker

### Recommended Scenarios

<Check>**PDF analysis** - Extract text, tables, or structure from documents</Check>
<Check>**Image interpretation** - Describe layouts, UI elements, charts, or diagrams</Check>
<Check>**Diagram analysis** - Explain relationships, flows, or architecture depicted</Check>
<Check>**Specific data extraction** - Pull particular information from visual content</Check>
<Check>**Context token optimization** - Need analyzed data, not entire raw file</Check>

### Avoid Multimodal Looker For

<Warning>**Plain text files** - Use `read` tool directly instead</Warning>
<Warning>**Source code** - Use `read` for exact contents needed for editing</Warning>
<Warning>**Files needing modification** - Looker only extracts, can't edit</Warning>
<Warning>**Simple file reading** - No interpretation needed, use regular tools</Warning>

## How It Works

Multimodal Looker follows a focused 4-step process:

1. **Receive request** - Gets file path and specific extraction goal
2. **Read and analyze** - Deeply interprets the visual content
3. **Extract target information** - Returns ONLY what was requested
4. **Pass to main agent** - Main agent continues work without processing raw file

### Key Principle

**Context token efficiency**: The main agent never processes the raw media file. Looker extracts and summarizes, saving thousands of tokens.

## When NOT to Use

Multimodal Looker is specialized - avoid using it when simpler tools work:

<Warning>
  **Source code files**: Use `read` tool directly. Looker interprets and summarizes, but editing requires exact original content.
</Warning>

<Warning>
  **Plain text documents**: No interpretation needed - regular `read` is faster and more accurate.
</Warning>

<Warning>
  **Files you'll modify later**: Looker's extracted summary can't be edited back into the original file.
</Warning>

<Warning>
  **When you need the full file**: If the entire content is needed, not just specific data, use `read` directly.
</Warning>

## Best Practices

<Check>**Be specific about what to extract** - Clear goals produce better results</Check>
<Check>**Request structured output** - Ask for tables, lists, or specific formats</Check>
<Check>**One file at a time** - Focused analysis is more accurate</Check>
<Check>**Use for token optimization** - When you need data from large PDFs</Check>

<Warning>**Don't expect perfection** - Vision models may misread text or miss details</Warning>
<Warning>**Don't use for critical exact data** - If precision is critical, verify manually</Warning>
