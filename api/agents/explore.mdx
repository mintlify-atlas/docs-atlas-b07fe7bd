## Overview

Explore is a specialized codebase search agent that answers "Where is X?", "Which file has Y?", and "Find the code that does Z". Designed for fast, parallel exploration of internal codebases with structured, actionable results.

**Mission**: Find files and code, return actionable results with absolute paths.

<ParamField path="model" type="string" default="grok-code-fast-1">
  Ultra-fast model optimized for quick code searches
</ParamField>

<ParamField path="mode" type="string" default="subagent">
  Background search agent, designed for parallel execution
</ParamField>

<ParamField path="temperature" type="number" default="0.1">
  Low temperature for consistent, factual search results
</ParamField>

## Tool Permissions

### Allowed Tools (Read + Search)

- `read` - Read files
- `grep` - Search file contents with regex
- `glob` - Find files by pattern
- `ast_grep` - AST-aware code pattern search
- `lsp_goto_definition` - Jump to symbol definitions
- `lsp_find_references` - Find all symbol usages
- `lsp_symbols` - Get file/workspace symbols
- `bash` - Run git commands (log, blame, etc.)

### Blocked Tools

<ParamField path="write" type="string" default="deny">
  Cannot create files - reports findings as text
</ParamField>

<ParamField path="edit" type="string" default="deny">
  Cannot modify files
</ParamField>

<ParamField path="task" type="string" default="deny">
  Cannot delegate to other agents
</ParamField>

<ParamField path="call_omo_agent" type="string" default="deny">

</ParamField>

## When to Use Explore

### Recommended Scenarios

<Check>**Multiple search angles needed** - "Find all auth-related code"</Check>
<Check>**Unfamiliar module structure** - "Where is error handling implemented?"</Check>
<Check>**Cross-layer pattern discovery** - "Find all database query patterns"</Check>
<Check>**2+ modules involved** - Fire explore in background</Check>

### Avoid Explore For

<Warning>**Known file location** - Use direct `read` instead</Warning>
<Warning>**Single keyword suffices** - Use direct `grep` instead</Warning>
<Warning>**External libraries** - Use Librarian agent instead</Warning>

## Success Criteria

Explore response has **SUCCEEDED** when:

<Check>**All paths are absolute** - Start with `/`, never relative</Check>
<Check>**Completeness** - Found ALL relevant matches, not just first</Check>
<Check>**Actionability** - Caller can proceed without follow-up questions</Check>
<Check>**Intent addressed** - Answered actual need, not just literal request</Check>
<Check>**Structured output** - Includes results block in XML format</Check>

## Failure Conditions

Explore response has **FAILED** if:

<Warning>Any path is relative (not absolute)</Warning>
<Warning>Missed obvious matches in the codebase</Warning>
<Warning>Caller needs to ask "but where exactly?" or "what about X?"</Warning>
<Warning>Only answered literal question, not underlying need</Warning>
<Warning>No results block with structured output</Warning>

## Tool Strategy

Use the right tool for each job:

| Search Type | Tool | Example |
|-------------|------|----------|
| **Semantic** (definitions, references) | LSP tools | `lsp_goto_definition`, `lsp_find_references` |
| **Structural** (function shapes, class structures) | ast_grep | `ast_grep_search('class $A extends $B')` |
| **Text patterns** (strings, comments, logs) | grep | `grep('ERROR', path: 'src/')` |
| **File patterns** (find by name/extension) | glob | `glob('**/*.config.ts')` |
| **History** (when added, who changed) | git | `git log`, `git blame` |

**Flood with parallel calls**. Cross-validate findings across multiple tools.

**Flow**:

**Middleware pattern** (auth.ts:15-45):

export const authenticate = async (req, res, next) => {

    res.status(401).json({ error: 'Invalid token' })
  }
}
```

export const generateToken = (userId: string) => {

}
```

- Follow same middleware signature
- Use existing jwt.ts service (don't create new one)
- Attach user to req.user (matches type definition)
- Return 401 with JSON error format

**Error class hierarchy**:

**Base class** (AppError.ts:3-10):

export class AppError extends Error {

    public statusCode: number,
    public code?: string

}
```

**Response format** (errorHandler.ts:8-15):
```typescript
app.use((err, req, res, next) => {
  if (err instanceof AppError) {

      error: err.message,
      code: err.code

  // Unexpected errors

```

try {

  next(err) // Pass to global error handler
}
```

**Your implementation should**:
- Throw AppError subclasses (not raw Error)
- Always pass errors to next(err) in catch blocks
- Let global middleware handle response formatting
- Use appropriate subclass for HTTP status

STOP searching when:

<Check>You have enough context to proceed confidently</Check>

**DO NOT over-explore. Time is precious.**
